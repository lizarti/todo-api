TODO API

Endpoints

Método Endpoint Descrição
GET /api/todos Lista todos os todos
POST /api/todos Cria um novo todo
GET /api/todo/{id} Seleciona o todo com o id especificado
PUT /api/todo/{id} Atualiza o todo com o id especificado
DELETE /api/todo/{id} Exclui o todo com o id especificado

Headers

201

Atualizar um recurso com o id especificado

PUT /api/todos/{id}

Request

Headers

content-type: application/json

Body
{
    title: "Tarefa 3",
    created_at: "2018-9-27 11:43:32",
    updated_at: "2018-9-27 11:55:25",
    done: true,
    id: 3
}

Response

200

Headers

content-type: application/json

Body
{
    "title": "Tarefa 1",
    "created_at": "2018-9-27 11:43:32",
    "updated_at": "2018-9-27 11:55:25",
    "done": true,
    "id": 3
}

Embora o método seja PUT, você não precisa enviar todos os campos.
O recurso não terá sua estrutura modificada de acordo com os dados enviados. Isso quer dizer que ocorrerá somente uma modificação (alteração de valor) nas propriedades enviadas.
Ou seja, é como o método PATCH (pra essa API os métodos PATCH e PUT são a mesma coisa).
